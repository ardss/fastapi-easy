## 🏗️ 架构概览

FastAPI-Easy 采用**分层架构**和**适配器模式**：

```
┌─────────────────────────────────────┐
│         FastAPI Application         │
└─────────────────────────────────────┘
                  ↓
┌─────────────────────────────────────┐
│          CRUDRouter (核心)          │
│  - 路由生成                          │
│  - 请求验证                          │
│  - 响应序列化                        │
└─────────────────────────────────────┘
                  ↓
┌─────────────────────────────────────┐
│         Hook System (扩展)          │
│  - before_* / after_* hooks         │
│  - 自定义业务逻辑                    │
└─────────────────────────────────────┘
                  ↓
┌─────────────────────────────────────┐
│       ORMAdapter (数据层)           │
│  - SQLAlchemy / Tortoise / MongoDB  │
│  - 统一接口                          │
└─────────────────────────────────────┘
                  ↓
┌─────────────────────────────────────┐
│            Database                 │
└─────────────────────────────────────┘
```

## 🔑 核心设计原则

### 1. 单一职责
每个组件只负责一件事，易于理解和维护。

### 2. 开闭原则
对扩展开放，对修改关闭。通过 Hook 系统扩展功能。

### 3. 依赖倒置
依赖抽象而非具体实现。通过适配器模式支持多种 ORM。

### 4. 接口隔离
提供最小化的接口，避免不必要的依赖。
