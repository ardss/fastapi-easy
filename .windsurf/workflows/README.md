# Windsurf 工作流

FastAPI-Easy 项目的自动化工作流集合。

## 可用工作流

### 1. `/运行测试并修复`

自动运行测试并修复错误。

**用途**:
- 在提交前运行所有测试
- 自动修复简单的测试失败
- 生成测试报告

**调用方式**:
```
在 Cascade 中输入: /运行测试并修复
```

**功能**:
- 运行完整测试套件
- 分析测试结果
- 逐个修复失败的测试
- 生成修复总结

### 2. `/代码质量检查`

检查并改进代码质量。

**用途**:
- 检查代码格式 (Black)
- 检查 import 排序 (isort)
- 检查代码风格 (flake8)
- 检查类型注解 (mypy)

**调用方式**:
```
在 Cascade 中输入: /代码质量检查
```

**功能**:
- 运行所有代码质量检查
- 自动修复格式问题
- 显示需要手动修复的问题
- 生成质量报告

## 快速开始

### 提交前的完整检查

1. 修改代码
2. 在 Cascade 中输入: `/代码质量检查`
3. 在 Cascade 中输入: `/运行测试并修复`
4. 如果所有检查都通过，提交代码

### 仅运行测试

在 Cascade 中输入: `/运行测试并修复`

### 仅检查代码质量

在 Cascade 中输入: `/代码质量检查`

## 工作流位置

所有工作流文件存储在 `.windsurf/workflows/` 目录中：

```
.windsurf/
└── workflows/
    ├── README.md (本文件)
    ├── run-tests-and-fix.md (/运行测试并修复)
    └── code-quality-check.md (/代码质量检查)
```

## 最佳实践

1. **提交前运行工作流** - 确保代码质量
2. **一次修复一个错误** - 便于追踪和调试
3. **保留输出日志** - 用于文档和参考
4. **定期运行检查** - 保持代码质量

## 故障排查

### 工作流不显示

- 确保文件在 `.windsurf/workflows/` 目录中
- 确保文件名为 `*.md`
- 重启 Windsurf IDE

### 工作流执行失败

- 检查命令是否正确
- 确保所有依赖都已安装
- 查看 Cascade 的错误消息
