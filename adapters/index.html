    适配器概览 (Adapters Overview)¶

    fastapi-easy 通过适配器模式（Adapter Pattern）支持多种数据库后端。这意味着你可以使用统一的 API (CRUDRouter) 来操作不同的数据库。

    支持的适配器¶

    适配器 底层库 数据库支持 特点 文档
    SQLAlchemy sqlalchemy PostgreSQL, MySQL, SQLite, Oracle, SQL Server 行业标准，功能最强 查看文档
    Tortoise tortoise-orm PostgreSQL, MySQL, SQLite 类似 Django，简单易用 查看文档
    MongoDB motor MongoDB 高性能异步 NoSQL 查看文档
    SQLModel sqlmodel 同 SQLAlchemy 结合 Pydantic 与 SQLAlchemy 查看文档

    如何选择？¶

    新项目 (SQL)¶

    推荐 SQLModel 或 SQLAlchemy - SQLModel: 如果你喜欢简洁的代码，一个类同时作为 ORM 模型和 API Schema - SQLAlchemy: 如果你需要更强大的 ORM 功能和更好的生态系统

    新项目 (NoSQL)¶

    使用 MongoDB - 适合文档型数据 - 灵活的 Schema - 高性能异步操作

    习惯 Django¶

    使用 Tortoise ORM - API 设计类似 Django ORM - 学习曲线平缓 - 原生异步支持

    企业级应用¶

    SQLAlchemy 是最稳健的选择 - 成熟稳定，经过大量生产环境验证 - 强大的查询能力 - 丰富的插件生态

    版本兼容性¶

    Python 版本¶

    • 最低要求: Python 3.8+
    • 推荐版本: Python 3.10+

    FastAPI 版本¶

    • 最低要求: FastAPI 0.100+
    • 推荐版本: FastAPI 0.104+

    Pydantic 版本¶

    • 要求: Pydantic 2.0+
    • 注意: 不支持 Pydantic 1.x

    数据库驱动版本¶

    • SQLAlchemy: 2.0+（异步模式）
    • Tortoise ORM: 0.19+
    • Motor (MongoDB): 3.0+
    • SQLModel: 0.0.8+

    常见问题 (FAQ)¶

    Q: 为什么不支持 Gino 或 Ormar？¶

    A: 这些 ORM 的维护状态不够活跃，且与 Pydantic 2.0 的兼容性存在问题。我们选择专注于维护良好、社区活跃的 ORM。

    Q: 未来会支持更多 ORM 吗？¶

    A: 我们会根据社区需求考虑添加新的适配器。如果你需要特定的 ORM 支持，欢迎提交 Issue 或 Pull Request。

    Q: 可以同时使用多个适配器吗？¶

    A: 可以！你可以为不同的资源使用不同的适配器。例如，用户数据用 PostgreSQL，日志数据用 MongoDB。

    # 用户使用 SQLAlchemy
user_router = CRUDRouter(
    schema=User,
    adapter=SQLAlchemyAdapter(UserDB, get_pg_db)
)

# 日志使用 MongoDB
log_router = CRUDRouter(
    schema=Log,
    adapter=MongoAdapter(log_collection)
)

    Q: 如何贡献新的适配器？¶

    A: 请参考 开发文档。新适配器需要： 1. 继承 BaseORMAdapter 基类 2. 实现所有抽象方法 3. 提供完整的测试覆盖 4. 编写使用文档

    自定义适配器¶

    如果你需要支持其他数据库（如 Redis, Elasticsearch），可以继承 BaseORMAdapter 并实现以下方法：

    from fastapi_easy.backends.base import BaseORMAdapter

class MyCustomAdapter(BaseORMAdapter):
    async def get_all(self, filters, sorts, pagination):
        # 实现获取所有记录的逻辑
        ...

    async def get_one(self, id):
        # 实现获取单条记录的逻辑
        ...

    async def create(self, data):
        # 实现创建记录的逻辑
        ...

    async def update(self, id, data):
        # 实现更新记录的逻辑
        ...

    async def delete_one(self, id):
        # 实现删除单条记录的逻辑
        ...

    async def delete_all(self):
        # 实现删除所有记录的逻辑
        ...

    async def count(self, filters):
        # 实现计数的逻辑
        ...

    详细的适配器开发指南请参考 贡献文档。

    性能考虑¶

    数据库索引¶

    无论使用哪个适配器，都要为常用的查询字段添加索引：

    # SQLAlchemy 示例
class ItemDB(Base):
    __tablename__ = "items"
    id = Column(Integer, primary_key=True)
    name = Column(String, index=True)  # 添加索引
    category = Column(String, index=True)  # 添加索引
    price = Column(Float)

    连接池配置¶

    对于生产环境，务必配置合适的连接池大小：

    # SQLAlchemy 示例
engine = create_async_engine(
    DATABASE_URL,
    pool_size=20,          # 连接池大小
    max_overflow=10,       # 最大溢出连接数
    pool_pre_ping=True,    # 连接健康检查
)

    批量操作¶

    使用批量操作而非循环单条操作：

    # ❌ 不推荐
for item in items:
    await router.create(item)

# ✅ 推荐
await router.bulk_create(items)
